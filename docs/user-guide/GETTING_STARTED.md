# Getting Started with Ancore Wallet

Welcome to Ancore Wallet. This guide helps new users go from zero to their first successful payment while setting up a secure backup.

## Before You Begin

- Install the Ancore Wallet browser extension from your approved distribution source.
- Confirm you are using the correct network (Testnet for learning, Mainnet for real funds).
- Make sure your device is updated and protected with a system passcode.

## 1) Creating an Account

1. Open the extension popup and select **Create Account**.
2. Enter an account label (example: `Main Wallet`).
3. Create a strong wallet password:
   - At least 12 characters
   - Mix upper/lowercase letters, numbers, and symbols
   - Avoid reusing a password from another service
4. Confirm your password.
5. Save your generated recovery phrase in an offline location.

### Account Creation Checklist

- [ ] Password is unique and strong
- [ ] Recovery phrase is written down correctly
- [ ] Recovery phrase is stored offline in a safe place

## 2) Your First Transaction

1. From the home screen, choose **Send**.
2. Paste or scan the recipient address.
3. Enter a small test amount first.
4. Review the transaction details:
   - Recipient address
   - Asset and amount
   - Estimated network fee
5. Confirm and sign the transaction.
6. Track confirmation status in **Recent Activity**.

### First Transaction Best Practice

Always send a small test payment before sending a large amount to a new address.

## 3) Backing Up Your Wallet

Your recovery phrase is the primary backup. Anyone with this phrase can access your funds.

### Recommended Backup Process

1. Write the phrase on paper (do not screenshot).
2. Create a second offline copy.
3. Store copies in physically separate secure locations.
4. Periodically verify both copies are still readable.

### What Not to Do

- Do not store recovery phrases in email drafts.
- Do not share recovery phrases with support staff.
- Do not store phrases in plain text cloud notes.

## 4) New-User Validation Script (Documentation Testing)

Use this script when testing docs with first-time users:

1. Ask the user to create an account using only this guide.
2. Observe where they hesitate for more than 10 seconds.
3. Ask them to complete a first test transaction.
4. Ask them to explain backup steps in their own words.
5. Collect feedback:
   - Which step was unclear?
   - Which term was unfamiliar?
   - What help content was missing?

Record findings in your team tracker and update docs weekly while onboarding is active.
